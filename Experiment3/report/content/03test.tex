\section{性能测试}

\subsection{使用 OpenSSL 测试 SM4 算法性能}

为了给后续自实现版本提供一个参考基线，首先使用 OpenSSL 的 \texttt{speed} 工具测试 SM4-CBC 的吞吐量。该工具在内存中构造不同大小的缓冲区，并在固定时间内重复调用 EVP 接口，因此得到的是“算法实现级别”的吞吐量参考值，而不是文件 I/O 场景下的整体性能。

\begin{lstlisting}[style=Bash]
openssl speed -elapsed -evp SM4-CBC
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{figure/openssltest.png}
    \caption{OpenSSL 的 SM4-CBC 性能测试结果}
    \label{fig:openssltest}
\end{figure}

由图可知，OpenSSL 实现的吞吐量约为 64 MB/s。该结果仅作为参考基线，用于说明成熟密码库实现与实验自编代码之间的差异。

\subsection{未优化版本基线测试}

未优化版本保留逐字节 S 盒查表、循环移位和异或操作，作为后续所有优化实现的统一基线。代码见附录中的 \verb|sm4_original.c|。
本次实验测试过程分为两层：

\begin{enumerate}
    \item x86 主机环境结果：用于调试、验证和参考对照；
    \item QEMU 模拟的 RISC-V 环境结果：作为正文分析的主结果来源。
\end{enumerate}

\subsubsection{主机参考结果}

基线测试，结果如图~\ref{fig:sm4test} 所示。该部分结果主要用于说明原始实现能够正确运行，并给出原生执行条件下的初始吞吐量参考，不作为本实验结论的重点来源。

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{figure/original.png}
    \caption{未优化 SM4 算法在主机环境下的早期测试结果（参考）}
    \label{fig:sm4test}
\end{figure}
从该参考结果中可以看到，未优化版本明显低于 OpenSSL 的成熟实现。后续统一批量脚本采用更大的数据规模和重复轮数，以降低短时间测试误差。

\subsubsection{QEMU 模拟 RISC-V 平台测试结果}

在 RISC-V 平台上，首先对 original 版本在 O0--O3 四个编译优化级别下进行交叉编译和运行测试。终端截图结果如图~\ref{fig:sm4test_riscv_default} 所示。

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{figure/riscv_original.png}
    \caption{未优化 SM4 算法在 QEMU-RISC-V 环境下的测试结果}
    \label{fig:sm4test_riscv_default}
\end{figure}
从结果可以看出，original 版本在 QEMU-RISC-V 平台上的绝对吞吐量显著低于主机原生环境，这主要是因为 QEMU 需要对 RISC-V 指令进行动态翻译执行。但与此同时，随着编译优化级别提高，性能提升非常明显，说明编译器优化对 RISC-V 平台的基础实现质量有着决定性影响。
