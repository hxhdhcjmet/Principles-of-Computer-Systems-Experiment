实验四 基于RISC-V开发板的加密算法性能测试与加速比分析
一、实验内容
本次实验基于每组分配到的RISC-V架构开发板，围绕openHiTLS中SM3或SM4加密算法开展不同技术方案下的性能测试与加速比分析，需设计并说明至少两种不同方案，完成运行时间、吞吐率和加速比测试。

二、基本任务
1.	学习并掌握RISC-V架构的开发版（荔枝派、香橙派）的基本使用方法。
2.	学习openHiTLS加密算法的使用示例，能够正确运行SM3或SM4实例。
3.	记录各方案的运行时间，吞吐率以及加速比测试。
4.	结合技术方案差异、开发板硬件条件、系统环境和测试方法等，对实验结果进行分析。
5.	若某一技术方案依赖开发板不支持的指令扩展，应说明原因并选择替代方案。

三、考察重点
1.	RISC-V架构开发版（荔枝派、香橙派）的使用，加密算法示例程序的编译、运行与测试；
2.	penHiTLS 中 SM3/SM4 示例程序的使用与正确性验证。实验方案设计、性能差异原因分析。

四、注意事项
1. 实验报告需交代清楚实验的硬件软件环境。
五、参考方案
1.	SM4默认循环 && SM4循环展开
2.	SM4默认XBox查表 && XBox合并
3.	SM4默认轮函数连续异或 && 减少指令依赖（将连续写同一变量改为临时变量合并后一次写回）
4.	备选：默认字节序转换 && RISC-V B扩展rev8转换，但仅支持zbb的板卡，开发板无zbb时不支持测试。
5.	若开发板支持其他扩展，也可通过其他扩展方案进行测试。

按如下步骤可测试sm4示例，附录中介绍了使用sm3进行测试的修改方法。
荔枝派版本：
	1、更换镜像源
		sudo cp /etc/apt/sources.list /etc/apt/sources_init.list
		sudo vim /etc/apt/sources.list
		将sources.list中的内容全部删除后改为
		deb http://deb.debian.org/debian sid main
	 	deb http://deb.debian.org/debian unreleased main
 		deb-src http://deb.debian.org/debian sid main
		将sources.list保存后输入
		sudo apt update
2、安装依赖项：
		sudo apt install cmake make git g++-riscv64-linux-gnu gcc-riscv64-linux-gnu libsctp-dev
3、获取libboundscheck源码并编译
		mkdir work && cd work
		git clone https://gitee.com/openeuler/libboundscheck.git
		cd libboundscheck
		make CC=riscv64-linux-gnu-gcc
		sudo cp ./lib/libboundscheck.so /usr/local/lib

vim ~/.bashrc

在文件的最后面加上：

export LD_LIBRARY_PATH=/home/sipeed/work/libboundscheck/lib:$LD_LIBRARY_PATH

	然后保存退出
source ~/.bashrc

4、获取 openhitls源码并编译
		cd ~/work
		git clone --recurse-submodules https://gitcode.com/openhitls/openhitls.git
		cd openhitls
		mkdir build && cd build
		python3 ../configure.py
		cmake .. -DCMAKE_CXX_COMPILER=riscv64-linux-gnu-gcc 
		make -j4
		sudo make install
5、运行示例代码
		cd ~/work/openhitls/testcode/demo/
		mkdir build && cd build
		cmake .. -DCMAKE_CXX_COMPILER=riscv64-linux-gnu-gcc
		make 
		./sm4cbc

如选择对sm3进行测试，要在获取openhitls源码后(即git clone后)，找到openhitls\testcode\demo\hash.c文件
将原程序中的
    uint8_t expResult[] = {0xd5, 0xc3, 0x03, 0x15, 0xf7, 0x2e, 0xd0, 0x5f, 0xe5, 0x19, 0xa1, 0xbf, 0x75,
        0xab, 0x5f, 0xd0, 0xff, 0xec, 0x5a, 0xc1, 0xac, 0xb0, 0xda, 0xf6, 0x6b, 0x6b, 0x76, 0x95, 0x98,
        0x59, 0x45, 0x09};
修改为
    uint8_t expResult[] = {0x14, 0x08, 0x1b, 0x03, 0xc8, 0xd6, 0xa9, 0x97, 0x12, 0x2b, 0xfa, 0x2c, 0x87,
            0x47, 0x8d, 0xe9, 0xb5, 0x23, 0xff, 0x6c, 0x96, 0x1b, 0xaa, 0xd3, 0xe1, 0x86, 0x32, 0xf6, 0x4d,
            0xc1, 0x04, 0xb4};
	将原程序中的
	ctx = CRYPT_EAL_MdNewCtx(CRYPT_MD_SHA256);
修改为
ctx = CRYPT_EAL_MdNewCtx(CRYPT_MD_SM3);
最后运行时将
./sm2sign
修改为
./hash
即可
